# Contribuer au projet

## Workflow

1. Crée une branche par sujet.
2. Lie la branche à une issue quand c'est possible.
3. Vérifie le projet dans Unity avant d'ouvrir une pull request.

## Bonnes pratiques

- Ne commite pas les dossiers générés par Unity.
- Garde les changements orientés sur une seule tâche.
- Décris clairement le but, le contexte et les tests effectués dans la pull request.

## Revue

- Une pull request doit être relue avant fusion.
- Si une modification touche le gameplay ou la scène, ajoute des captures ou une courte note de validation.
